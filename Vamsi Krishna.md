# Project 1 · ADAS & LiDAR 2D/3D Annotation QA

> **Company:** Adwait Algorithm as (Avistos), Hyderabad  
> **Role:** Data Annotator  
> **Period:** July 2020 – July 2021  
> **Platform:** Amazon SageMaker · CVAT · Mtruck · ART Tool

---

## 📌 Overview

Performed 2D and 3D LiDAR spatial annotation and quality validation for **ADAS and autonomous driving datasets**. Labeled image and point-cloud data using bounding boxes, polygons, and cuboids to support object detection, depth estimation, and scene understanding models.

---

## 🧠 Annotation Scope

### LiDAR Point Cloud — 3D Cuboid Annotation

| Object Class | Annotation Type |
|---|---|
| Cars, Trucks | 3D Cuboid with heading/orientation |
| Pedestrians | 3D Cuboid with occlusion flags |
| Traffic Signs | 2D Bounding Box in camera frame |
| Cyclists | 3D Cuboid with size constraints |

**Key tasks:**
- Labeled point-cloud data using cuboids with accurate yaw orientation and spatial placement
- Annotated camera-fused image frames with 2D bounding boxes for sensor fusion pipelines
- Validated label consistency across consecutive LiDAR frames for temporal accuracy
- Classified transportation objects to support object detection, depth estimation, and scene understanding

---

### Video Object Tracking QA

- Segmented video footage into individual frames for frame-level annotation
- Tracked objects (humans, vehicles, packages, pets) using bounding boxes across frames
- Ensured non-truncated boxes, tight alignment, and consistent object IDs across sequences
- Maintained dataset reliability for downstream computer vision model training

---

## 🔧 QA Process

```
Step 1 → Annotation per guidelines (bounding box / cuboid / polygon)
Step 2 → Self-review for spatial accuracy and class correctness
Step 3 → Frame consistency check across video sequences
Step 4 → Dataset validation and QA audit before delivery
```

## 📌 Overview

Led quality assurance and dataset creation for a production-grade **Text-to-Speech (TTS) voice system** at Oracle. Responsibilities spanned TTS custom voice dataset curation, speech transcription validation, voice labeling, and linguistic quality assurance across multiple industry verticals to improve ASR/TTS model performance.

---

## 🧠 Evaluation Scope

### 1. TTS Custom Voice Dataset Creation

- Supported voice labeling, speech transcription validation, and TTS custom voice dataset creation
- Curated domain-specific TTS datasets and scripts across multiple industries
- Ensured linguistic accuracy and natural speech output for production ASR/TTS models

**Industry verticals covered:**

| Industry | TTS Focus Area |
|---|---|
| IT | Technical terminology, product and brand names |
| Healthcare | Medical terms, drug names, clinical language |
| Call Centres | Conversational scripts, IVR dialogue flows |
| Chatbots | Multi-turn dialogue, natural phrasing patterns |
| Voice Assessment | Pronunciation scoring, fluency evaluation |

---

### 2. Speech Transcription Validation

- Validated transcription accuracy against source audio
- Flagged mismatches in phonetic rendering, punctuation, and spoken-form conversion
- Reviewed edge cases: numbers, abbreviations, mixed-language inputs, proper nouns

**Sample validation categories:**

```
Input Type         | Common Issue Caught          | Corrective Action
-------------------|------------------------------|----------------------
Currency (₹1,500)  | Incorrect spoken form        | Corrected to "fifteen hundred rupees"
Dates (21/04/2024) | Literal slash rendering      | Corrected to "twenty-first April"
Abbreviations      | "Dr." read as "doctor period"| Corrected to "Doctor"
Mixed script text  | Garbled transliteration      | Flagged for re-recording
```

---

### 3. Voice Labeling & Audio QA

- Labeled audio samples with speaker metadata, language tags, and quality flags
- Reviewed recordings for background noise, clipping, mispronunciation, and unnatural prosody
- Maintained structured QA logs tracking defect type, severity, and resolution status

---

### 4. SOP & Guidelines Development

- Developed annotation guidelines and SOPs for TTS dataset creation workflows
- Standardized quality benchmarks adopted across the annotation team
- Collaborated with data scientists and ML engineers to refine dataset quality and improve model output
